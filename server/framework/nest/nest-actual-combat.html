1. 环境变量 ​

环境变量是在运行 nodejs 应用时，传递给 node 应用的一组参数，这些参数可以通过 process.env() 来获得程序相关的环境变量信息。

环境变量作为约定俗成的应用配置，具有以下作用：

  1. 程序可能会在不同的环境中运行，通过设置不同的环境变量，达到动态切换的作用。
  2. 根据 12-Factor 的第12条，更加推荐将应用的配置存储在环境变量中。
  3. 通常不将敏感信息（秘钥、密码、数据库连接账号密码等）硬编码到代码中，而是通过存储在环境变量的方式读取。

1.2. 配置模块 ​

ConfigModule 配置模块接收一个 options 选项对象，支持自定义读取 .env 配置文件，一般要求配置文件存放在项目的根文件夹。

假设我们想要将各种 .env 文件存储在 config 文件夹，只需要传递参数让 ConfigModule 作为动态模块导入，即可改变配置文件夹的存储位置。

编写 ConfigModule 动态模块 ​

  1. 配置模块接收一个可选参数，表示 env 文件的存储位置
  2. 动态模块注册，需要调用它的 forRoot/register 方法动态注册
  3. 读取 env 的配置，设置在 process 里面
  4. 配置服务，从 process 中读取环境变量

使用 ConfigurableModuleBuilder 使其注册为动态模块

ts
// src/config/config.module.ts
import { DynamicModule, Module } from '@nestjs/common';
import { ConfigService } from './config.service';
import { ConfigurableModuleClass } from './config.module-definition';
import { ConfigOptions } from './interfaces';
import * as path from 'path';
import * as fs from 'fs';
import * as process from 'process';
import * as dotenv from 'dotenv';
import * as _ from 'lodash';

@Module({
  providers: [ConfigService]
})
export class ConfigModule extends ConfigurableModuleClass {
  static forRoot(options?: ConfigOptions): DynamicModule {
    // 1.读取env配置文件，如未传递文件名称，默认以 root/.env 作为配置文件名称
    const envFilename = !options?.envFilename ? '.env' : options.envFilename;
    const filepath: string = !fs.existsSync(options.folder)
      ? path.resolve(process.cwd(), envFilename)
      : path.resolve(options.folder, envFilename);

    try {
      // 2.将env配置文件的内容设置到process.env中
      const configData = fs.readFileSync(filepath, 'utf-8');
      const envConfig = dotenv.parse(configData);
      this.assignVariablesToProcess(envConfig);
    } catch (e) {
      console.log(e);
    }

    return {
      ...super.forRoot(options)
    };
  }

  /**
   * 将env配置文件的key-value合并到process.env中
   * @param envConfig 配置文件的key-value
   * @private
   */
  private static assignVariablesToProcess(envConfig: Record<string, unknown>) {
    if (!_.isObject(envConfig)) {
      return;
    }

    const keys = Object.keys(envConfig).filter((key) => !(key in process.env));

    keys.forEach((key) => {
      const value = envConfig[key];

      if (typeof value === 'string') {
        process.env[key] = value;
      } else if (typeof value === 'boolean' || typeof value === 'number') {
        process.env[key] = `${value}`;
      }
    });
  }
}

使用可配置的模块构建器 ConfigurableModuleBuilder 创建动态模块

ts
// src/config/config.module-definition.ts
import { ConfigurableModuleBuilder } from '@nestjs/common';

export interface ConfigOptions {
  folder: string;
  envFilename?: string;
}

export const { ConfigurableModuleClass, MODULE_OPTIONS_TOKEN } = new ConfigurableModuleBuilder<ConfigOptions>()
  .setClassMethodName('forRoot')
  .setExtras({ isGlobal: true }, (definition, extras) => ({
    ...definition,
    global: extras.isGlobal
  }))
  .build();

自定义逻辑读取env配置文件并测试读取

ts
// src/config/config.service.ts
import { Injectable } from '@nestjs/common';
import * as process from 'process';

@Injectable()
export class ConfigService {
  constructor() {}

  get(key: string): string {
    if (!(key in process.env)) {
      return undefined;
    }

    return process.env[key];
  }
}
